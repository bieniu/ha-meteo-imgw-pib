# Instructions for AI Agents (Copilot, Claude, Codex)

## Repo overview
- Custom Home Assistant integration for Meteo IMGW-PIB (Polish meteorological data API).
- Integration code in `custom_components/meteo_imgw_pib/`. Tests in `tests/`.

```
custom_components/meteo_imgw_pib/
├── __init__.py          # Entry point (async_setup_entry / unload)
├── manifest.json        # HA integration metadata
├── const.py             # Domain, config keys, update interval
├── config_flow.py       # UI-based config flow (select from station list)
├── coordinator.py       # DataUpdateCoordinator with type-annotated ConfigEntry
├── entity.py            # Base entity (CoordinatorEntity subclass)
├── sensor.py            # Sensor platform
├── weather.py           # Weather platform (proxy-only, forecast twice_daily + hourly)
├── diagnostics.py       # Diagnostics
├── icons.json           # Icon mappings
└── translations/        # en.json, pl.json
```

## Dev environment (uv)
- Python 3.14 required. Venv lives in `./.venv`.
- `scripts/setup-local-env.sh` creates it: `uv venv --python 3.14 && source .venv/bin/activate && uv sync && uv run prek install`
- `uv sync` resolves dependencies from `pyproject.toml` / `uv.lock`. No `requirements-*.txt`.
- CI runs `uv sync --frozen` — lockfile is the source of truth.

## Commands (run from venv or via `uv run`)
| Task | Command |
|------|---------|

## HA integration conventions
- Target: Python 3.14, HA 2026.7+.
- I/O must be async. Blocking work → `hass.async_add_executor_job`.
- Handle errors with precise HA exceptions (`ConfigEntryNotReady`, `ConfigEntryAuthFailed`, `HomeAssistantError`). No bare `except` outside config flow / background tasks.
- Logging: lazy (`%s`), no trailing periods, no sensitive data.
- User-facing text: American English, friendly, second person, sentence case, backtick for names.
- Type `ConfigEntry` with a `runtime_data` dataclass alias (see `coordinator.py`).
- Similar reference integrations: `accuweather`, `met`, `open_meteo`.
- Reference docs:
  - <https://developers.home-assistant.io/docs/core/entity/sensor>
  - <https://developers.home-assistant.io/docs/core/entity/weather>

## Docstrings & comments
- Docstrings: prefer one-liners. If needed, max 2–3 lines — never longer.
- Comments explain *why* something is done, not *what* the code does. Do not restate the code or add comments for obvious decisions.

## Testing
- `pytest` with `pytest-homeassistant-custom-component`. Use `MockConfigEntry`, snapshot assertions (`syrupy`), and `init_integration` helper (`tests/__init__.py`).
- Snapshots stored in `tests/snapshots/` (amber format, custom extension re-roots to `<test_dir>/snapshots`).
- Mock all external APIs via the `mock_imgw_pib_client` fixture (patches `ImgwPib` client). `conftest.py` provides realistic `WeatherData`.
- Test through integration setup (`init_integration`), not by instantiating entities directly.
- Config flow: 100% coverage required. Test all paths (user, abort, error, duplicate).

## Code reviews
- Do not `amend`, `squash`, or `rebase` during a review.
- PR descriptions and responses to review comments must be written by a human, not generated by AI.
